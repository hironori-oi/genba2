📋 テストの進め方

Step 1. URL を開いてログイン

携帯ブラウザで https://genba2-ai.vercel.app/login を開き、worker でログイン。

Step 2. scan-first フローへ

以下のいずれかの URL を直接開くと、即 scanner が起動します（HTTPS なのでカメラ permission が降ります）:
  • /app/logi/receiving?mode=scan — 受入
  • /app/logi/picking?mode=scan — ピッキング
  • /app/logi/inventory?mode=scan — 棚卸
  • /app/works/manufacturing?mode=scan — 製造
または /app/logi/ トップから「スキャンで開始」 secondary CTA を tap。

Step 3. カメラ permission 許可

ダイアログで「許可」。HTTPS なので iOS Safari / Android Chrome ともに OK。

Step 4. 下記の QR を試し読み

PC 画面の QR にカメラを向け、画面の中央枠内に収めて 1-2 秒静止。
各 QR で ResultOverlay の表示（成功 / 失敗 / fallback to ManualInput）を確認。

Step 5. 訂正 flow も試す

worker でも到達可能な /app/correct/movements 等で、自分が登録した record を訂正 → 2 段確認ダイアログの挙動確認（特に /app/correct/manufacturing の rollback_inflow=true 時）。

Step 6. 個人設定で i18n / dark mode

/app/account/preferences で language=en、theme=dark に切替 → 全画面適用確認。

Step 7. (tenant_admin) admin 画面ウォークスルー

tenant_admin で再ログインし、/app/admin の 12 cards を順に確認。特に新規:
  • /app/admin/qr-formats — version clone + 2 段確認ダイアログ
  • /app/admin/csv-formats — 50-row column mapping editor
  • /app/admin/work-settings — 4 業務 tab
  • /app/admin/reports — recharts dashboard (daily/weekly/monthly)
  • /app/admin/audit-logs — フィルタ + CSV export
  • /app/admin/usage — monthly_scan_usage

Step 8. 印刷 preview

/app/logi/history → 「印刷」launcher で 4 帳票を試す。A4 / 80mm thermal toggle 動作確認。

⚠️ 困った時

  • カメラ permission が出ない: ブラウザ設定で genba2-ai.vercel.app の権限をリセット、ページ reload
  • QR が認識されない: 距離 15-30cm、明るさ十分、QR を画面中央枠内
  • ログイン失敗: synthetic user 削除済の可能性 → owner の実アカウントで login
  • scan 後何も起きない: 該当 tenant の qr_format_definitions が空 → /app/admin/qr-formats で format 登録（tenant_admin で）
  • build error / 503: Vercel デプロイ中の可能性、3-5 分待って再 access
  • カメラ起動して数秒で止まる: Phase 3a Scanner の自動 fallback 動作（BarcodeDetector 未対応ブラウザで ManualInput modal 表示）
🚨 production data に注意: synthetic test tenant (11d4774b-cba2-4d7a-9c98-0eeab4334161) 内では作成・削除自由、ただし owner の実テナントで操作すると本番データに影響します。実テナント login 時は read-only walkthrough 推奨。
